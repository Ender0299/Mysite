sito hostato su una vps con ddns e apache(programma che riconduce ip a file e cartelle)

sito web centos = - installare httpd(yum install httpd); fare start e enable di httpd(systemctl start httpd,systemctl enable httpd) - controllare status(systemctl status httpd);
                    abilitare http e https nel firewall e renderli statici(firewall-cmd --add-service=http, firewall-cmd --add-service=https, firewall-cmd --runtime-to-permanent) - controllare status (firewall-cmd --list-all);
                    caricare file index.html con collegamento ftp
                  - RENDERLO ONLINE = avere un ddns e aprire porte 80 e 443 sul router
                  - (ssl valido) Abilitare le epel-release(apt install epel-release); installare moudlo per apache(apt install mod_ssl); installare certbot(apt install certbot); nstallre la versione per apache(apt install python3-certbot-apache);
                    creare un virtual host privato dove poter mettere i nuovi certificati (cd /etc/httpd/conf.d , nano yourDomainName.conf); scrivere il virtualhost( <VirtualHost *:80>; ServerName yourDomainName.com; DocumentRoot /var/www/html; </VirtualHost> );
                    controlare se viene rilevato (httpd -D DUMP_VHOSTS); installa i certificati(certbot --apache)
                    [https://stackoverflow.com/questions/59549309/unable-to-find-a-virtual-host-listening-on-port-80-please-add-a-virtual-host]


sito web ubuntu = - installare apache (apt install apache2); fare start ed enable (systemctl start apache2,systemctl enable apache2) - controllare status(systemctl status apache2);
                    caricare file index.html con collegamento ftp
                  - RENDERLO ONLINE = avere un ddns e aprire porte 80 e 443 sul router
                  - (Self-singed) Intallare openssl(apt install openssl); creare file key per il cripting https (openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout /etc/ssl/private/nome che vuoi.key -out /etc/ssl/certs/nome che vuoi.crt);
                    impostare i certificati nel file (nano /etc/apache2/sites-available/default-ssl.conf); impostare il redirect sul protocollo https (nano /etc/apache2/sites-available/000-default.conf);
                    abilitare il firewall con eccezione (ufw allow 'Apache Full') ; abilitare il certificato ssl (a2enmod ssl, a2ensite default-ssl) ; fare un restart del servizio (systemctl restart apache2)
                    [https://www.digitalocean.com/community/tutorials/how-to-create-a-self-signed-ssl-certificate-for-apache-in-ubuntu-18-04]
                  - (ssl valido) Abilitare le epel-release(apt install epel-release); installare moudlo per apache(apt install mod_ssl); installare certbot(apt install certbot); installre la versione per apache(apt install python3-certbot-apache);
                    creare certificati (certbot --apache)


connessione ftp = scaricare vsftpd e configurarlo(nano /etc/vsftpd.cof)

connessione ssh ubuntu = scaricare openssh -apre le porte(22) in ipv4 in automatico
